使用Clion进行STM32标准库开发

发表于2023-11-13|更新于2023-11-30|嵌入式
|阅读量:

环境

  • ArchLinux
  • STM32CubeMX
  • arm-none-eabi-gcc
  • OpenOCD
  • Clion
  • stlink

安装

sudo pacman -S arm-none-eabi-gcc openocd stlink
paru -S stm32cubemx clion
#若出现编译不通过的情况，安装arm-none-eabi-newlib
sudo pacman -S arm-none-eabi-newlib

CLion配置

  • 打开设置-构建、执行、部署-嵌入式开发
  • 分别选择OpenOCD和Stm32CubeMX位置
  • 打开设置-构建、执行、部署-工具链
  • 选择编译器
  • 新建嵌入式项目
  • 跳转到CubeMX后选择芯片并覆盖原文件

CubeMX

  • 输入Project Name,注意和Clion的工程名、路径一致
  • 选择Toolchain/IDE为SW4STM32
  • 根据需求选择堆栈大小
  • 关闭CubeMX

Stlink

在工程中添加配置文件

MyConfig.cfg:

# choose st-link/j-link/dap-link etc.
# adapter driver cmsis-dap
# transport select swd
source [find interface/stlink.cfg]
transport select hla_swd
source [find target/stm32f1x.cfg]
# download speed = 10MHz
adapter speed 10000

标准库移植

用keil标准库模板中的Core和Driver替换生成的文件，并添加启动文件等
